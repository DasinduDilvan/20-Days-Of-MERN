# 🚀 20-Days-Of-MERN

A focused 20-day MERN stack learning + building challenge where every concept learned is immediately applied to a real project.

Instead of traditional “learn first, build later” approach, this roadmap follows:

> 📚 Learn → 🛠️ Build → 🚀 Apply instantly

By the end of 20 days, you will have:
- Strong MERN fundamentals (MongoDB, Express, React, Node.js)
- A working full-stack project
- Authentication system (JWT)
- CRUD functionality
- Deployment experience
- Real-world development workflow

---

# 🎯 Goal

To become capable of building and deploying a **complete MERN application** within 20 days by learning and implementing daily.

This is not just learning — this is **building every single day**.

---

# ⚙️ Tech Stack

- Frontend: React (Vite)
- Backend: Node.js + Express
- Database: MongoDB + Mongoose
- Authentication: JWT + bcrypt
- API Communication: Axios
- Deployment: (Render / Vercel / Netlify / MongoDB Atlas)

---

# 📅 20-Day Learning + Building Roadmap

| Day    | Learn Topic 1                                         | Learn Topic 2                           | Immediately Apply to Your Project                                         |
| ------ | ----------------------------------------------------- | --------------------------------------- | ------------------------------------------------------------------------- |
| **1**  | ES6+ (let, const, arrow functions, template literals) | Array Methods (`map`, `filter`, `find`) | Create project folder, initialize Git, build sample data and render lists |
| **2**  | Objects & Destructuring                               | Modules (`import` / `export`)           | Organize project files and reusable utility functions                     |
| **3**  | Async JavaScript (`Promise`, `async/await`)           | Fetch API & Axios                       | Connect to a dummy/public API and display data                            |
| **4**  | React Setup (Vite)                                    | JSX & Components                        | Create project structure with Header, Navbar, Footer, Home                |
| **5**  | Props                                                 | `useState`                              | Build navigation and interactive UI (buttons, toggles, cards)             |
| **6**  | `useEffect`                                           | Rendering Lists & Conditional Rendering | Load and display dynamic content                                          |
| **7**  | React Forms                                           | Controlled Components                   | Build Register and Login pages                                            |
| **8**  | React Router                                          | Layout Components                       | Add multi-page navigation (Home, About, Dashboard, Login)                 |
| **9**  | Axios in React                                        | Loading & Error Handling                | Connect frontend to a temporary/mock backend                              |
| **10** | Node.js Basics                                        | npm & Environment Variables             | Create backend project and configure `.env`                               |
| **11** | Express.js                                            | Routing                                 | Build your first REST API endpoints                                       |
| **12** | Middleware                                            | Express Error Handling                  | Add logging, request validation, and centralized error responses          |
| **13** | MongoDB                                               | Mongoose Schemas & Models               | Connect your database and create your first collection                    |
| **14** | CRUD Operations                                       | Postman Testing                         | Build Create, Read, Update, Delete APIs and verify them                   |
| **15** | Connect React ↔ Express                               | Axios CRUD Integration                  | Display, create, edit, and delete real data from the frontend             |
| **16** | JWT Authentication                                    | Password Hashing (`bcrypt`)             | Implement secure Signup and Login                                         |
| **17** | Protected Routes                                      | Authorization (User Roles if needed)    | Protect dashboard and private API routes                                  |
| **18** | Image/File Upload                                     | Cloud Storage (or local uploads first)  | Add profile image or project image upload                                 |
| **19** | Search & Filtering                                    | Pagination or Sorting                   | Improve usability with search and filters                                 |
| **20** | Deployment (Frontend + Backend + Database)            | Environment Variables in Production     | Deploy the complete application and test it online                        |

# 📌 How to Use This Roadmap

### 🔁 Daily Rule
- Learn the concepts
- Implement immediately in the project
- Commit code to GitHub daily

### 💡 Core Principle
> If you learn something today, it must appear in your project today.

---

# 🏗️ Project Strategy

Start building your hackathon project from **Day 4 onward**, not at the end.

Every new concept should upgrade your project step-by-step.

Example:
- Learn Forms → Build Registration page
- Learn JWT → Add Login system
- Learn MongoDB → Store user data
- Learn Express → Build APIs
- Learn Deployment → Publish app
